# 销售数字分身（Sales Digital Twin）技术方案

**版本**：v1.0  
**运行时**：Python 3.12  
**LLM**：DeepSeek（OpenAI 兼容 API）  
**框架**：LangChain + Pydantic v2

---

## 1. 方案概述

整体采用 **自定义 Agent Workflow 固定流水线编排**，底层依托 LangChain 提供 LLM 调用与结构化输出解析能力。

**选型依据**：

1. 业务流程为固定线性流水线：分类校验 → 信息提取 → 建议生成 → 结果聚合
2. PRD 已明确每步逻辑与输出规范，无需大模型自主规划
3. LangChain 仅承担 LLM 接口调用、Pydantic 结构化解析两项能力
4. 流程编排、分支短路、分流判断由自研 Pipeline 显式硬编码控制
5. 自定义 Agent = 单一职责 LLM 任务单元，区别于通用自主规划 Agent

## 2. 系统架构

### 2.1 流水线总览

```
Input Text
    │
    ▼
[RuleEngine 关键词预筛] ──高置信 non_sales──► 兜底响应
    │
    ▼
[ClassifierAgent] ──non_sales──► 兜底响应
    │
    sales
    ▼
[ExtractorAgent] ──► ExtractionResult
    │
    ├──► [FormatterService] ──► 四行文本
    ├──► [CRMAdvisorAgent] ──► CRM 建议
    └──► [ContractAdvisorAgent] ──► 合同建议
    │
    ▼
PipelineResult → Console + outputs/*.txt
```

### 2.2 四层分层架构

**分层顺序**：表现层 → 应用层 → 领域层 → 基础设施层

**依赖流向**：`cli` → `core/pipeline` → `agents/services` → `infrastructure`

| 层级 | 目录 | 职责 |
| --- | --- | --- |
| 表现层 | `cli/` | CLI 参数解析、文件/stdin 读取、控制台打印、结果落盘 |
| 应用层 | `core/` | Pipeline 编排、Factory 依赖注入、常量、异常、路径管理 |
| 领域层 | `agents/` `services/` `models/` | Agent 推理、规则引擎、格式化、强类型数据模型 |
| 基础设施层 | `infrastructure/` | 配置、DeepSeek 客户端、Prompt 加载、重试、日志 |

**约束**：下层模块无感知上层业务逻辑；Agent 间只传递 Pydantic 模型，禁止裸字典跨模块传递。

## 4. 核心模块设计

### 4.1 Agent 职责

| Agent | 职责 | 输入 | 输出 |
| --- | --- | --- | --- |
| ClassifierAgent | 场景分类复核 | `text` | `ClassificationResult(scene)` |
| ExtractorAgent | 结构化信息提取 | `text` | `ExtractionResult` |
| CRMAdvisorAgent | CRM 字段变更建议 | `text, extracted` | `CRMSuggestion` |
| ContractAdvisorAgent | 合同变更建议 | `text, extracted` | `ContractSuggestion` |

### 4.2 确定性服务（无 LLM）

| 服务 | 职责 |
| --- | --- |
| RuleEngine | 销售关键词预筛，高置信场景直接分流 |
| FormatterService | 模板渲染固定四行文本，格式 100% 可控 |

### 4.3 Pipeline 编排逻辑

```python
def process(self, request: ProcessRequest) -> PipelineResult:
    text = request.text.strip()

    # Step 1: 场景分类（规则 + LLM 复核）
    scene = self._classify(text)
    if scene is SceneType.NON_SALES:
        return PipelineResult.fallback("非销售场景，跳过处理")

    # Step 2: 信息提取
    extracted = self.extractor.run(text)

    # Step 3: 格式化 + 双 Agent 建议
    formatted = self.formatter.format(extracted)
    crm = self.crm_advisor.run(text, extracted)
    contract = self.contract_advisor.run(text, extracted)

    # Step 4: 聚合输出
    return PipelineResult.success(formatted, crm, contract)
```

## 5. 场景分类策略

```
Input Text
    │
    ▼
RuleEngine 关键词计数
    ├── ≥3 命中 → sales（高置信，跳过 LLM）
    ├── 0 命中  → non_sales（高置信，零 LLM 调用）
    └── 1~2 命中 → ClassifierAgent LLM 复核
```

**销售关键词示例**：价格、合同、报价、签约、合作、需求、方案、付款、交付、项目、订单、客户、商机、采购

## 6. 数据模型

### 6.1 核心 Schema

```python
class NextAction(BaseModel):
    person: str = "待确认"
    task: str = "待确认"
    deadline: str = "待确认"

class ExtractionResult(BaseModel):
    customer_name: str = "待确认"
    summary: str
    pain_points: list[str] = []
    next_action: NextAction

class CRMFieldUpdate(BaseModel):
    field: str
    old_value: str
    new_value: str

class CRMSuggestion(BaseModel):
    updates: list[CRMFieldUpdate]

class ContractSuggestion(BaseModel):
    has_change: bool
    suggestion: str

class PipelineResult(BaseModel):
    is_fallback: bool
    fallback_message: str | None = None
    formatted_text: str | None = None
    crm_text: str | None = None
    contract_text: str | None = None
```

### 6.2 数据流转

```
str → ProcessRequest → SceneType → ExtractionResult
  → CRMSuggestion + ContractSuggestion → PipelineResult → str
```

## 7. Agent 基类与 LLM 集成

所有 Agent 继承 `BaseAgent`，统一 **Prompt 加载 → LLM 调用 → 结构化解析 → 重试**：

```python
class BaseAgent(ABC, Generic[T]):
    prompt_name: str
    output_model: type[T]

    def run(self, **kwargs) -> T:
        return self._invoke_with_retry(kwargs)

    def _build_chain(self):
        prompt = self._prompt_loader.load(self.prompt_name)
        return prompt | self._llm.with_structured_output(self.output_model)
```

## 8. 关键设计决策

| 决策 | 选择 | 原因 |
| --- | --- | --- |
| 编排方式 | 自研 Pipeline 硬编码 | 流程固定，输出稳定，易测试 |
| 分类策略 | 规则 + LLM 双层 | 降本、非销售快速短路 |
| CRM 集成 | 仅建议输出 | 符合作业约束，避免数据污染 |
| 合同模块 | 独立 Agent | 语义域不同，Prompt 更精准 |
| 输出格式 | 文本为主 | 符合作业固定格式要求 |
| 人机协同 | 人工确认后写入 | 避免 LLM 幻觉污染 CRM |

## 9. 容错与稳定性

| 场景 | 策略 |
| --- | --- |
| API 超时/限流 | 指数退避重试，最多 2 次 |
| JSON/Schema 解析失败 | 重试 1 次，仍失败抛 `AgentOutputError` |
| 字段缺失 | Pydantic 默认值「待确认」 |
| 空输入 | 立即抛 `EmptyInputError`，不调 LLM |
| 非销售高置信 | 规则短路，零 LLM 调用 |

**异常体系**：`SalesDigitalTwinError` → `EmptyInputError` / `AgentOutputError` / `LLMInvokeError` / `ConfigurationError`
