## 1. 整体技术架构方案

整体架构采用自定义 Agent Workflow 固定流水线编排方案，底层依托 LangChain 提供 LLM 基础工具能力支撑。

## 2. 选型核心依据

1. 业务流程为固定线性流水线，标准执行顺序：分类校验 → 信息提取 → 建议生成 → 结果聚合；
2. PRD 已明确每一步执行逻辑、入参、输出规范，无需大模型自主规划步骤、动态循环调用工具；
3. LangChain 仅做底层基础能力封装，仅承担两项能力：LLM 接口调用、基于 Pydantic 的结构化输出解析；
4. 流程编排、分支短路、分流判断逻辑全部由自研 Pipeline 代码显式硬编码控制，执行链路固定，输出稳定性高，方便本地演示与单元测试；
5. 本项目自定义 Agent 定义：单一职责 LLM 任务单元，仅执行单次专项文本推理；区别于通用智能 Agent（自主思考、多轮规划、动态工具调用）。

## 3. 四层分层架构（自上而下）

**分层顺序：** 表现层 → 应用层 → 领域层 → 基础设施层

**依赖流向规范：** `cli` → `core/pipeline` → `agents/services` → `infrastructure`

**约束：** 下层模块无感知上层业务逻辑，各层级完全解耦。

### 3.1 表现层 `cli/`

仅负责交互输入输出，无任何业务推理逻辑：

- 命令行参数解析
- 测试用例文本读取加载
- 最终结果控制台打印输出
- 推理结果文件持久化落盘存储

### 3.2 应用层 `core/`

业务流程总调度层，统一组装所有领域组件，串联完整执行链路：

- 完整 Pipeline 流水线编排逻辑
- 依赖注入工厂 Factory
- 全局业务常量统一定义
- 自定义业务异常类
- 项目全局文件路径统一管理

### 3.3 领域层

承载核心业务逻辑，分为三大子模块：

#### （1）`agents/`：4 个单一职责 LLM Agent 单元

| Agent 名称 | 职责说明 |
| --- | --- |
| ClassifierAgent | 场景分类复核大模型单元 |
| ExtractorAgent | 客户沟通信息结构化提取单元 |
| CRMAdvisorAgent | CRM 更新建议生成单元 |
| ContractAdvisorAgent | 商业合同变更建议生成单元 |

#### （2）`services/`：确定性规则服务（无 LLM 调用，纯代码逻辑）

- **RuleEngine**：销售场景关键词预筛规则引擎
- **FormatterService**：标准四行文本固定格式化渲染服务

#### （3）`models/`：全链路 Pydantic 强类型数据模型

- 组件、Agent 间流转数据统一使用结构化实体
- 禁止裸字典、原始 LLM 字符串跨模块传递

### 3.4 基础设施层 `infrastructure/`

底层通用基础能力，为上层提供统一工具封装：

- `.env` 环境配置读取管理
- DeepSeek 大模型 API 客户端封装
- 外置独立 Prompt 文本加载管理器
- 网络请求重试机制封装
- 全局统一日志工具
- API / 业务统一异常捕获处理

## 4. 完整业务执行流水线流程

### 4.1 入口预处理

接收原始通话文本字符串，封装为 `ProcessRequest` 结构化请求模型；前置执行 `RuleEngine` 关键词预筛（无 LLM 调用，低成本快速分流）：

1. 命中销售关键词 ≥ 3 个：标记 `sales` 销售场景，直接进入完整 LLM 推理链路；
2. 命中销售关键词 = 0 个：标记 `non_sales` 非销售场景，直接短路返回兜底文案，全程不调用大模型；
3. 命中销售关键词 1~2 个：边界模糊样本，进入 `ClassifierAgent` 大模型二次分类复核。

### 4.2 分类分支分流

1. LLM 复核判定为 `non_sales`：直接输出兜底文案，流程终止；
2. LLM 复核判定为 `sales` 销售对话：进入信息提取流程。

### 4.3 结构化信息提取

调用 `ExtractorAgent`，输出强类型实体 `ExtractionResult`，包含字段：客户名称、沟通摘要、客户需求、下一步行动。

### 4.4 标准化文本格式化

调用 `FormatterService`，通过模板确定性渲染标准四行文本，不依赖大模型，输出格式 100% 合规可控。

### 4.5 并行生成两类业务建议

并行执行双 Agent 推理，分别输出结构化结果：

- `CRMAdvisorAgent` → `CRMSuggestion`
- `ContractAdvisorAgent` → `ContractSuggestion`

### 4.6 结果聚合输出

1. 整合提取信息、格式化文本、两类建议，聚合为统一总结果实体 `PipelineResult`；
2. 双渠道统一输出：
   - 控制台打印完整标准化文本；
   - 自动写入 `outputs` 目录文本文件，持久化保存推理结果。
